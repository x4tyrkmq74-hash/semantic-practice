Basic Text Tags

  • p – paragraph text, usually multiple lines
  • em – emphasized text (adds meaning)
  • strong – important text (adds meaning)
  • cite – title of a work
  • mark – highlighted/marked text
  • b – attention text (not meaning-based)
  • i – alternative voice
  • u – underlined text
  • kbd – keyboard input
  • samp – sample output
  • var – variable name

When choosing a tag, think about what your text actually is—paragraph, address, code sample, etc.

HTML tags shouldn’t be used just for styling. For example, don’t use strong or b to make text bold unless it has meaning.

Tags for Grouping and Describing Content

  • h1 – main page title (only one per page)
  • h2 – subtopic under an h1
  • h3 – subtopic under an h2
  • h4 – subtopic under an h3
  • h5 – subtopic under an h4
  • h6 – subtopic under an h5

section elements group related content and must include a heading.

Headings work like an outline—main topic, subtopic, supporting items.

Special Characters

  • © – copyright symbol
  •   – non-breaking space
  • & – ampersand
  • < and > – angle brackets

These characters help avoid breaking HTML and let you safely display symbols.

Lists

  • ol – ordered list
  • ul – unordered list
  • li – list item inside a list

Lists help organize content and improve accessibility for screen-reader users.

You can nest lists to show relationships between items, such as headings and subheadings.
